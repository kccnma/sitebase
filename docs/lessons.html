Site Base Lessons

A step-by-step tutorial to learn the basics of front-end development by manually hand-coding a simple framework.

Site Base 3.0 Lessons

Site Base 3.0 builds upon Site Base 2.0 by using Gulp, a task manager, to handle SASS pre-processing via a command line interface (CLI). Designed for those looking to learn how to set up a more advanced front-end development workflow.

Site Base 2.0 Lessons

Site Base 2.0 builds upon Site Base 1.0 by adding support for SASS. Designed for those with a basic understanding of HTML, CSS, and JS. No command line (CLI) required. New software is required to process SASS (e.g. in these lessons we will use PrePros, a GUI-based compiler.

Site Base 1.0 Lessons

Site Base 1.0 is a static HTML + CSS + JS base (so precompiler required/no SASS). The lessons below are for those interested in coding a basic front-end framework for a simple static site from scratch by hand, line-by-line. No prior web design or coding experience required. Designed for first-time front-end coders, Site Base 1.0 uses no pre-processing (e.g. SASS), no command line interface (CLI), and no special software. All you need is a text editor and a browser.

Step 1 Base Blank

Description:

Set up your local development environment via a basic folder and file structure.

  1. Create a "base-blank" root folder
  2. Create "css" and "js" sub folders
  3. Add an index.html file
    • Add starter markup
  4. Link to a style.css file
    • Add starter css table of contents
    • Add HTML5 support styling
  5. Link to a script.js file

Step 2 Base Content

Description:

Set up your base typography along with essential UI components (e.g. buttons) and media (e.g. responsive images).

  1. Duplicate your "base-blank" folder and rename it to "base-content"
  2. Delete your "js" folder and link to it (we will add it back later)
  3. Add base content markup to your index.html file and base content styles to your style.css file:
    • Base Typography
      • Body, Headings, Paragraphs, Links, Lists, Blockquotes, & Helpers
    • Base Media
      • Images
    • Base Components
      • Buttons

Step 3 Base Layout

Description:

Make your base responsive by setting up media queries targeting different screen sizes and use a simple grid for the layout of your content.

  1. Duplicate your "base-content" folder and rename it to "base-layout"
  2. Remove all content from index.html and add "demo" grid markup
  3. Add media queries and a simple grid to your style.css file
    • Media Queries (with body font-size settings and a "container" that is fixed width for large screens and fluid for small screens)
    • Custom Grid (multiple columns for large screens, stacked columns for small screens)

Step 4 Base Site Structure

Description:

Use semantic HTML5 elements to set up a common site structure including a site header, footer, navigation, and main content area.

  1. Duplicate your "base-layout" folder and rename it to "base-site"
  2. Remove all content from index.html and add base structural site markup:
    • Header, Footer, Navigation, Main Content Areas (e.g. Hero Section, 3-Column placeholder content)
  3. Add base structural site styling your style.css file
    • Base Site Structure
      • Header, Logo, Footer, Sections
    • Base Navigation
      • Header Nav, Footer Nav

Step 5 Mobile Menu (Toggle Nav)

Description:

Use Javascript to make a a simple toggle-style main navigation for small screen sizes.

  1. Add back a "js" folder and script.js file
  2. Add a "menu" element to your main navigation.
  3. Add a custom JavaScript function that will toggle a class to open and close the main navigation on smaller screens.
  4. Style the menu to look like a three-bar navicon.

Step 6 Sub Page(s)

Description:

Create sub pages that are well suited for specific content, such as a two-column sub page with a main content area and sidebar with a sub nav. Other sub page examples include an about/article/post page, a post/product/image listing page, a product/feature detail page, and contact page with form.

  1. Update the index.html page site navigation to link to the following example sub pages;
    • about.html, products.html, product-detail.html, contact.html
  2. Copy/paste the index.html page and to create each of the above example sub pages
  3. Potential New Features/components:
    • inline page anchors, accordians (collapsible panels), responsive grid for lists and thumbnail images, articles, forms

Step 7 Custom Site

Override the base styling with site-specific styling and custom HTML, CSS, & JS.

  1. Add a section entitled "SITE SPECIFIC CSS" to the bottom of your base style.css file and use this area to add any custom site-specific (e.g. project-specific, client-specific, etc) styling. Try not to edit your "BASE CSS" above. Instead, try to override the base styles at the bottom of your style sheet in this designated area that is seperate from your base styles.
  2. Style (CSS) changes to consider:
    • Customize the typography.
    • Use a custom color palette
    • Consider adding aesthetic graphical elements such as line, shape, texture, and background photography.
  3. Content (HTML) changes to consider:
    • Change the header and brand identity of the site (e.g. change the site name, add a logo, add a tagline, etc.).
    • Use real content (no placeholder photos or copy).
    • Consider using a "content-first" approach to establish a strong content hierarchy, information architecture (IA), and navigation.
    • Use your grid (rows and columns) to experiment with different layouts to best present the content.
    • Utilize and embrace known good usability practices, UX design patterns, and UI components.
